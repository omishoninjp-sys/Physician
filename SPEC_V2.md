## 1. 一頁要旨

**何をするプロダクトか：**
税理士と顧問先（経営者・経理担当）が、決算申告に必要な書類のやり取りを「**構造化されたワークフロー**」で行う共同ワークスペース。

**なぜ作るか：**
現状の Email/LINE/Dropbox/紙ベースの書類回収は混乱の温床。税理士の最大の時間ロスは、AI 仕訳ではなく**「資料を集める作業」**。ここに焦点を絞る。

**Google Drive とは何が違うか：**
- 必要書類が**範本ベースで構造化**されている
- 顧問先が**何を出せばいいか明確**にわかる
- 提出・確認・完了の**進捗が見える**
- 確認後**ロック**で証跡が残る
- 「最新版どれ？」問題が**消える**

**誰に売るか：**
税理士事務所（B2B2B モデル）。1 事務所契約で 50〜500 件の顧問先がぶら下がる。

## 2. 商品定位

### 2.1 ターゲット
- **主**：中小型税理士事務所（顧問先 30〜300 件規模）
- **副**：独立税理士・税理士法人内のチーム

### 2.2 競合との位置取り

| 軸 | Google Drive | 弥生 / freee の顧客 portal | **GoyouLink v2** |
|---|---|---|---|
| 業界特化 UX | × 汎用 | △ 半業界特化 | ✓ 税務専用 |
| ガイド付き checklist | × | × | **✓ 中核** |
| 確認後ロック | × | × | **✓ 中核** |
| 進捗管理 | × | △ | **✓ 中核** |
| client-facing 体験 | △ | △ | **✓ 中核** |

### 2.3 価値方程式

税理士目線：
- 資料回収にかかる時間 50% 減
- 「最新版どれ？」問題が消える
- 監査時の証跡管理が自動化される
- 顧客あたりの担当可能件数が増える → 事務所収益増

顧問先目線：
- 「何を出せばいい？」が明確
- 進捗が見える
- 同じことを何度も聞かれない
- 過去資料の一元保管

## 3. 核心ワークフロー（7 段階）

```
┌────────────────────────────────────────────────────────────────┐
│ Phase 1: 初期ヒアリング                                          │
│   税理士が顧問先の状況を口頭確認                                  │
│   （第一期？不動産取引あり？借入あり？等）                         │
└─────────────────────────┬──────────────────────────────────────┘
                          ▼
┌────────────────────────────────────────────────────────────────┐
│ Phase 2: 書類リクエスト                                          │
│   税理士が範本一覧から必要なものをチェック                         │
│   → 顧問先側にリアルタイム反映                                   │
└─────────────────────────┬──────────────────────────────────────┘
                          ▼
┌────────────────────────────────────────────────────────────────┐
│ Phase 3: 書類提出                                                │
│   顧問先がドラッグ＆ドロップでアップロード                         │
│   → 即座に「提出済」表示、税理士に通知                            │
└─────────────────────────┬──────────────────────────────────────┘
                          ▼
┌────────────────────────────────────────────────────────────────┐
│ Phase 4: 税理士レビュー                                          │
│   税理士が確認 → OK or 差し戻し                                  │
│   （差し戻しの場合は理由付きで顧問先に通知）                       │
└─────────────────────────┬──────────────────────────────────────┘
                          ▼
┌────────────────────────────────────────────────────────────────┐
│ Phase 5: 暫定決算書の作成・送付                                  │
│   税理士が暫定決算書をシステム経由で送付                          │
└─────────────────────────┬──────────────────────────────────────┘
                          ▼
┌────────────────────────────────────────────────────────────────┐
│ Phase 6: 顧問先確認（ロック）                                     │
│   経営者が確認・承認 → **永久ロック**                            │
│   - 顧客側は以後変更不可                                          │
│   - 税理士は変更可（変更時は再確認フローへ）                       │
│   - 確認時の snapshot hash を audit log に記録                   │
└─────────────────────────┬──────────────────────────────────────┘
                          ▼
┌────────────────────────────────────────────────────────────────┐
│ Phase 7: 申告書交付・アーカイブ                                  │
│   税理士が申告書一式をアップロード                                │
│   → 顧問先はいつでもダウンロード可                                 │
│   → 過年度参照として永続保存                                      │
└────────────────────────────────────────────────────────────────┘
```

## 4. 範本構造（15 件、43 個 templates）

5 種類の頻度タイプによって UI 動作が完全に異なる：

### 4.1 永続/変動時（4 件）
**ルール：** 一度提出すれば永久有効、変動時のみ再提出。

| # | 範本名 |
|---|---|
| 1 | 登記簿謄本 |
| 2 | 定款 |
| 13 | 賃貸契約書 |
| 14 | 固定資産の契約書 |

### 4.2 一次性（設立時のみ、1 親 + 6 子）
**ルール：** 新設法人のみ。提出後は完全に隠す。

3. 開業時届出書
- 設立届（国）
- 設立届（都道府県）
- 設立届（市町村）
- 青色申告届出書
- 源泉所得税納期特例
- 給与支払事務所開設届

### 4.3 過年度参照（2 親 + 18 子）
**ルール：** 旧事務所からの onboard 時のみ手動提出。以後は自動で過去 period から参照。

4. 前期の申告書一式（国税・地方税・消費税・内訳書・決算書・固定資産台帳・試算表・総勘定元帳・仕訳日記帳）
5. 前々期の申告書一式（同上 9 項目）

### 4.4 当期毎（毎期収集、6 件）
**ルール：** 毎会計年度 reset、再提出必要。

| # | 範本名 |
|---|---|
| 7 | 銀行口座の明細（+ 合計記帳明細） |
| 8 | 経費領収書 |
| 9 | クレジットカードの明細 |
| 10 | 売上明細等 |
| 11 | 給与明細 |
| 12 | 借入金の返済予定表（有借入時） |

### 4.5 随時（変動時のみ、1 親 + 3 子）
**ルール：** 顧問先側から能動的に上げる。phase 紐付けなし。

6. 異動届（国・都道府県・市町村）
15. 税務署 / 県税事務所 / 市役所から送られてくる封書

## 5. 主要機能（v2 MVP scope）

### 5.1 税理士側

**A. 顧問先一覧 dashboard**
- 担当顧問先のリスト
- 各社の当期進捗（提出済/未提出/レビュー待ち）
- 期限警告

**B. 書類リクエスト**
- 範本リストから必要項目をチェック
- カテゴリ別 collapse 表示
- 親範本は一括選択可能
- リアルタイムで顧問先に反映

**C. 書類レビュー**
- 提出された書類を確認
- OK / 差し戻し（理由付き）
- 自動ステータス更新

**D. 決算書送付**
- 暫定決算書をアップロード
- 顧問先確認依頼

**E. 申告書交付**
- 最終申告書を提出
- 自動アーカイブ

### 5.2 顧問先側

**A. 書類提出 portal**
- 何を出せばいいかの一覧（税理士が選択したもののみ）
- ドラッグ＆ドロップアップロード
- 該当なし宣言ボタン

**B. 進捗確認**
- 自分が出すべきもの／出したもの／確認待ち が一目でわかる

**C. 決算書確認・承認**
- 暫定決算書を確認
- 承認ボタン（ワンクリックで永久ロック）

**D. 申告書ダウンロード**
- 過去申告書をいつでも DL

### 5.3 共通機能

- 全操作の audit log（7 年以上保存）
- ファイル hash 改ざん検知
- メール/LINE 通知
- 検索（年度別、書類種類別、ステータス別）

## 6. 顧客 onboarding 流程

### 6.1 既存法人を onboard する場合
1. 税理士が顧問先を登録
2. 「過年度参照」範本（前期・前々期申告書）を手動アップロード（前事務所から引き継いだもの）
3. 「永続」範本（登記簿・定款・契約書類）をアップロード
4. 当期の period 作成 → 「当期毎」範本を顧問先にリクエスト

**所要時間目標：30 分以内**

### 6.2 新設法人を onboard する場合
1. 税理士が顧問先を登録
2. 「設立時届出書」範本（6 件）の有無確認・アップロード
3. 「永続」範本（登記簿・定款）アップロード
4. 当期 period 作成 → 「当期毎」範本リクエスト

**所要時間目標：20 分以内**

## 7. データモデル概要

```
tenants（税理士事務所）
  └── users（税理士・スタッフ）
  └── clients（顧問先）
       └── client_users（経営者・経理担当）
       └── periods（会計年度）
            └── document_requests（範本毎の請求）
                 └── document_submissions（実ファイル）
                      └── confirmations（顧問先確認・ロック）

document_categories（global、7 件）
  └── document_templates（global、43 件）
       └── tenants が override 可能（事務所別カスタマイズ）
```

詳細は別途「システム架構規劃書 v2」（次回作成）。

## 10. 対象外範囲（明示的に scope out）

以下は **v2 では作らない**：

- ❌ 仕訳・会計帳簿機能（既存の弥生・freee が圧倒的）
- ❌ AI 仕訳（市場は既に飽和。後回し）
- ❌ 給与計算・社保（業界別ツール多数あり）
- ❌ 顧客の経理代行
- ❌ 申告書の自動作成（税理士の業務領域）

GoyouLink v2 は **「書類フロー専用」**、税理士の本来業務には踏み込まない。

## 12. 想定リスク・対策

| リスク | 対策 |
|---|---|
| 税理士が既存 ad-hoc 方式（Excel + Dropbox + LINE）から移行しない | **1 日以内に 1 顧問先を onboard できる UX**、移行ツール提供 |
| 「ただの Dropbox」化 | フォルダではなく**範本 + ステータス**を UI の主役に |
| 顧問先が UI を使いこなせない | **シンプル UI 死守**、必要最小限のアクション |
| 法規対応漏れ | 電帳法対応の hash、タイムスタンプ、ログを day-1 から実装 |
| セキュリティ事故 | 監査ログ、二要素認証、最小権限原則、暗号化 |
